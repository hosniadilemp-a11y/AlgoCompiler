Comment le score est calculé ?

Le classement est construit pour rester compétitif mais aussi équitable quand peu de personnes ont encore résolu le problème. Le score final va de 0 à 40.

1. Les 4 composantes du score

  • Tests : 10 points si le problème est entièrement résolu.
  • Temps de résolution : plus vous résolvez vite, plus la note est haute.
  • Temps d'exécution moyen : plus votre solution s'exécute vite, plus la note monte.
  • Mémoire moyenne : moins votre solution consomme, plus la note est élevée.

Important : seules les soumissions qui ont réussi tous les tests entrent dans ce leaderboard.

2. La formule utilisée

Étape A : on calcule une note brute entre 2 et 10 selon votre position entre le meilleur et le moins bon solveur.
note brute = 2 + 8 × position relative
Étape B : on mesure le nombre de participants pour éviter des écarts trop sévères quand le leaderboard est petit.
confiance = min(1, (participants - 1) / 4)
Étape C : on rapproche la note brute vers un centre neutre à 6 quand il y a peu de participants.
note finale = 6 + confiance × (note brute - 6)

3. Pourquoi le meilleur utilisateur peut avoir seulement 7 ou 8 ?

Quand il n'y a que 2 participants, la confiance vaut 0.25. Le système évite donc de donner immédiatement 10 au premier et une note très basse au second. Le meilleur tourne autour de 7 et l'autre reste proche de 5.

Quand il y a 3 participants, la confiance vaut 0.50. Le meilleur se rapproche alors de 8, le milieu d'environ 6 et le dernier d'environ 4.

4. Cas particuliers

  • Si tous les solveurs ont exactement la même valeur pour une métrique, tout le monde reçoit 10 pour cette métrique.
  • Si une ancienne soumission n'a pas encore de données d'exécution ou de mémoire, elle reçoit une valeur minimale de 2 pour cette métrique.
  • Le score affiché quand vous survolez la pastille bleue détaille toujours les 4 composantes utilisées.
